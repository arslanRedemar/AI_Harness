# AI Instruction Writing Guide (AIWG)

AI에게 전달되는 지침을 작성하는 절차 및 원칙.

---

## 핵심 원칙

| 원칙 | 정의 | 위반 예시 | 준수 예시 |
|------|------|-----------|-----------|
| 간결성 | 미사여구 없이 핵심만 기술 | "매우 중요하므로 반드시 꼭 확인해야 합니다" | "확인 필수" |
| 구조성 | 리스트·표·페어 형식 사용 | 산문형 설명 나열 | 번호 목록 또는 표 |
| 구체성 | 주어·목적어 명시, 용어 일관성 | "그것을 처리한다" | "API 응답 JSON을 파싱한다" |

---

## 작성 절차

### 1. 목적 정의
- 한 문장으로 지침의 목적을 작성한다.
- 형식: `[대상]이 [행동]하기 위한 지침`
- 예: `LLM이 코드 리뷰를 수행하기 위한 지침`

### 2. 맥락 최소화
- 제공하는 맥락을 필요 최소한으로 제한한다.
- 각 맥락 항목에 대해 "이것이 없으면 AI가 작업을 수행할 수 없는가?"를 자문한다.
- NO → 제거한다.

### 3. 구조 선택

| 내용 유형 | 권장 형식 |
|-----------|-----------|
| 순서가 있는 절차 | 번호 목록 |
| 순서 없는 항목 | 불릿 목록 |
| 대응 관계 | 2열 표 (키-값) |
| 조건 분기 | `if/else` 블록 또는 표 |
| 단일 사실 | 한 줄 서술 |

### 4. 용어 표준화
- 동일한 개념에는 동일한 단어를 사용한다.
- 동의어 혼용 금지: `사용자 / 유저 / 고객` → 하나로 통일.
- 약어는 첫 등장 시 한 번만 정의한다: `PRD(Product Requirements Document)`

### 5. 검토 체크리스트

- [ ] 주어가 모든 문장에 명시되어 있다.
- [ ] 목적어가 모호한 지시어(`그것`, `이를`)를 사용하지 않는다.
- [ ] 같은 개념에 두 가지 이상의 단어를 쓰지 않는다.
- [ ] 감탄사·수식어·강조 표현이 없다.
- [ ] 산문 단락이 없다 (모두 목록·표·단문으로 분해되었다).
- [ ] 맥락 없이도 의미가 통하는 항목을 제거했다.

---

## 안티패턴

| 패턴 | 문제 | 대안 |
|------|------|------|
| 강조 중첩 | "매우 중요하고 반드시 필수적으로" | "필수:" 레이블 사용 |
| 암묵적 주어 | "확인한 후 처리한다" | "AI가 입력값을 확인한 후 변환한다" |
| 과도한 예시 | 3개 이상의 유사 예시 나열 | 대표 예시 1개 |
| 중의적 조건 | "필요한 경우 실행" | "입력값이 null이면 실행" |
| 배경 서술 | 히스토리·이유 설명 포함 | 행동 지침만 기술 |

---

## 형식 템플릿

```
# [지침 이름]

목적: [대상]이 [행동]하기 위한 지침.

## 입력
- [입력 항목 1]: [타입/형식]
- [입력 항목 2]: [타입/형식]

## 규칙
1. [규칙 1]
2. [규칙 2]

## 출력
- [출력 항목 1]: [타입/형식]

## 예외
- [조건]: [처리 방법]
```

---

## 참고

- 맥락 크기와 AI 성능은 반비례한다. 지침은 짧을수록 좋다.
- 사람이 읽기 편한 문체(서술형, 감정적 강조)는 AI 성능을 저하시킨다.
- 구조가 없는 지침은 AI가 자체적으로 구조를 추론하게 만들어 토큰을 낭비하고 의도에서 벗어난 결과를 초래한다.
